## XSS

禁止直接渲染不可信 HTML。谨慎使用：

```js
dangerouslySetInnerHTML
v-html
innerHTML
```

必须使用时确认：

- 内容来源可信，或已进行清洗。
- 使用白名单过滤。
- 不允许脚本执行。
- 不把用户输入直接拼入 HTML。

## 权限控制

涉及权限时检查：

- 页面权限
- 路由权限
- 菜单权限
- 按钮权限
- 接口权限
- 数据权限
- 角色权限
- feature flag

不要仅靠前端隐藏按钮作为权限控制。前端只负责体验和可见性，关键权限必须以后端或既有权限体系为准。

## 敏感信息

禁止硬编码：

- Token
- Secret
- API Key
- 私钥
- 内部接口地址
- 用户隐私数据

应使用环境变量或项目已有配置系统。不要把敏感信息写入日志、埋点、错误提示、localStorage 或截图。

## 外链安全

外链使用新窗口打开时，考虑：

```html
rel="noopener noreferrer"
```

同时检查跳转来源是否可信，避免开放重定向和钓鱼风险。

## 表单安全

表单提交需要考虑：

- 输入校验
- XSS 风险
- 重复提交
- 权限校验
- 错误提示
- 敏感字段展示
- 文件上传类型和大小限制

校验逻辑应复用项目既有表单和 schema 方案。

## 错误监控

项目已有监控系统时，关键异常应考虑上报：

- 接口失败
- 核心流程失败
- 页面渲染异常
- 支付或提交失败
- 关键异步任务失败

错误提示遵循项目已有 toast、message、notification、fallback UI 或 error boundary。

## 性能监控

性能敏感页面关注：

- FCP
- LCP
- CLS
- INP
- TTFB

不要为了埋点引入明显性能开销。

## 埋点

新增埋点前检查：

- 是否已有同类事件。
- 命名是否统一。
- 参数是否符合规范。
- 是否重复触发。
- 是否涉及隐私数据。

不要重复埋点，不要记录敏感字段。
